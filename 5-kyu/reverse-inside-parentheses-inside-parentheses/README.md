<h2><a href=https://www.codewars.com/kata/5e07b5c55654a900230f0229/train/javascript target="_blank">Reverse Inside Parentheses (Inside Parentheses)</a></h2><h3>5 kyu</h3><p>In this kata, you will be given a string of text and valid parentheses, such as <code>"h(el)lo"</code>. You must return the string, with only the text inside parentheses reversed, so <code>"h(el)lo"</code> becomes <code>"h(le)lo"</code>. However, if said parenthesized text contains parenthesized text itself, then that too must reversed back, so it faces the original direction. When parentheses are reversed, they should switch directions, so they remain syntactically correct (i.e. <code>"h((el)l)o"</code> becomes <code>"h(l(el))o"</code>). This pattern should repeat for however many layers of parentheses. There may be multiple groups of parentheses at any level (i.e. <code>"(1) (2 (3) (4))"</code>), so be sure to account for these.</p><p>For example:</p><pre><code class="language-javascript"><span class="cm-variable">reverseInParens</span>(<span class="cm-string">"h(el)lo"</span>) <span class="cm-operator">==</span> <span class="cm-string">"h(le)lo"</span>;<span class="cm-variable">reverseInParens</span>(<span class="cm-string">"a ((d e) c b)"</span>) <span class="cm-operator">==</span> <span class="cm-string">"a (b c (d e))"</span>;<span class="cm-variable">reverseInParens</span>(<span class="cm-string">"one (two (three) four)"</span>) <span class="cm-operator">==</span> <span class="cm-string">"one (ruof (three) owt)"</span>;<span class="cm-variable">reverseInParens</span>(<span class="cm-string">"one (ruof ((rht)ee) owt)"</span>) <span class="cm-operator">==</span> <span class="cm-string">"one (two ((thr)ee) four)"</span>;</code></pre><p>Input parentheses will always be valid (i.e. you will never get <code>"(()"</code>). Blank string will also be considered as valid, and should return blank string.</p>
