<h2><a href=https://www.codewars.com/kata/595467c63074e38ba4000063/train/javascript target="_blank">Simple Fun #333: Incomplete Virus</a></h2><h3>5 kyu</h3><h1 id="task">Task</h1><p>John is a hacker. He succeeded in invading a server. </p><p>His next job is to write <code>n</code> numbers(1...n) in the memory of the server. He did this by using a virus code he wrote.</p><p>Unfortunately, since the server only accepts binary digits, and his virus code doesn't take into account the problem, only a fraction of the numbers are written into memory.</p><p>Given a number <code>n</code>, your task is to calculate how many numbers are written into memory.</p><p>Note, <code>n</code> may be very huge, so, it will be given by a string.</p><h1 id="example">Example</h1><p>For <code>n = "10"</code>, the output should be <code>2</code>.</p><p>2 number <code>"1"</code> and <code>"10"</code> are written into memory.</p><p>For <code>n = "20"</code>, the output should be <code>3</code>.</p><p>3 number <code>"1"</code>, <code>"10"</code> and <code>"11"</code> are written into memory.</p><p>For <code>n = "100"</code>, the output should be <code>4</code>.</p><p>4 number <code>"1"</code>, <code>"10"</code>, <code>"11"</code> and <code>"100"</code> are written into memory.</p>
