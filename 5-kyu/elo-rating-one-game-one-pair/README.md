<h2><a href=https://www.codewars.com/kata/55633765da97b266e3000067/train/javascript target="_blank">Elo rating - one game, one pair</a></h2><h3>5 kyu</h3><h1 id="task">Task</h1><p>Calculate a chess player's new Elo rating using their previous ratings (<code>experience</code>), their opponent's rating (<code>opponent</code>), the outcome of the new game (<code>score</code>), and the Factor k function (<code>k</code>).</p><h2 id="arguments">Arguments</h2><ul><li><code>experience</code> is an array/list containing the history of the player's ratings, with the last item in the array being the current rating. For example, if <code>experience</code> is <code>[995, 1025, 1050]</code>, then the player's 2 old ratings are 995 and 1025, and their current rating is 1050. If a player has no recorded experience (<code>experience = []</code>), use 1000 for their rating.</li><li><code>opponent</code> is a single number indicating the opponent's rating</li><li><code>score</code> is the outcome of the new game. <code>score</code> is <code>1</code> if the player won, <code>0</code> if the player lost, or <code>0.5</code> if it was a tie.</li><li><code>k</code> is a function that takes in the <code>experience</code> array/list as an argument. <code>k</code> is an optional argument - sometimes a <code>k</code> function will be passed for you to use and sometimes not. You must write a default <code>k</code> function to use in case no <code>k</code> function is passed (see below for details).</li></ul><h2 id="return">Return</h2><ul><li>The <code>elo</code> function must return the new rating of the player rounded to the nearest integer.</li></ul><h1 id="elo-rating-formula">Elo rating formula</h1><p>The Elo rating system starts with an expectation formula:</p><pre><code>expectation = 1 / (1+10^((opponent_rating - player_rating)/400))</code></pre><p>This can then be used in the new rating formula as follows:</p><pre><code>new_player_rating = player_rating + k(experience) * (score - expectation)</code></pre><p>Note that <code>k</code> is a function taking in <code>experience</code> as a parameter.</p><h2 id="factor-k-function">Factor K function</h2><p>You must write a default <code>k</code> function to use if no <code>k</code> function is passed to the <code>elo</code> function. The default <code>k</code> function should take in <code>experience</code> as an argument and return the following:</p><ul><li>If player is a newbie (less than 30 games of experience), return 25.</li><li>If player is not a newbie, but player's rating has never reached 2400 before, return 15.</li><li>If player is not a newbie and has at least one record of a rating &gt;= 2400, return 10.</li></ul><p>(These are pre-2011 FIDE rules.)</p><h1 id="examples">Examples</h1><ul><li>Example 1: <code>elo([], 1725, 1)</code></li></ul><p>Since player has no experience, we will use 1000 as their rating. <code>opponent</code> rating is 1725.<code>score</code> is 1.</p><p>Then we have:</p><pre><code>expectation= 1 / (1+10^((opponent_rating - player_rating)/400))= 1 / (1+10^((1725 - 1000)/400))= 0.01516572425000013</code></pre><p>Since no <code>k</code> function was passed, we will use our default <code>k</code> function. Since player is a newbie (less than 30 games of experience), <code>k(experience)</code> will return 25.</p><p>Then in our new_player_rating formula:</p><pre><code>new_player_rating= player_rating + k(experience) * (score - expectation)= 1000 + 25 * (1 - 0.01516572425000013)= 1024.62085689375</code></pre><p>When we round this to the nearest integer our final answer is 1025.</p><ul><li>Example 2:</li></ul><pre><code>def k_algo(exp):    return 800//len(exp)    elo([1025, 1025, 1050, 1075, 1100, 1125, 1150,1175, 1200, 1225, 1250, 1275, 1300, 1325, 1350,1375, 1400, 1425, 1450, 1475, 1500, 1525, 1550,1575, 1600, 1625, 1650, 1675, 1700, 1725], 1000,0, k_algo)</code></pre><p>The player's current rating is the last item in experience, 1725. <code>opponent</code> rating is 1000.<code>score</code> is 0.A function called <code>k_algo</code> is passed which we will use instead of our default <code>k</code> function.</p><p>Then we have:</p><pre><code>expectation = 1 / (1+10^((opponent_rating - player_rating)/400))= 1 / (1+10^((1000 - 1725)/400))= 0.9848342757499998</code></pre><p>When we pass <code>experience</code> to the <code>k_algo</code> function, we will get out <code>800//len(experience) = 26</code>.</p><p>Then in our new_player_rating formula:</p><pre><code>new_player_rating = player_rating + k(experience) * (score - expectation)= 1725 + 26 * (0 - 0.9848342757499998)= 1699.3943088305</code></pre><p>When we round this to the nearest integer our final answer is 1699.</p>
