<h2><a href=https://www.codewars.com/kata/52cc2bbafb9a9985a700061f/train/javascript target="_blank">Number.prototype.toDecimal</a></h2><h3>5 kyu</h3><p>Numbers in JavaScript can be turned into strings. For example, <code>(123.456).toString()</code> gives you the string <code>"123.456"</code>. But sometimes, numbers are so large or so small that the string returned is in a strange format called "scientific notation".</p><p>Here are some examples of scientific notation, as seen in JavaScript:</p><pre><code class="language-javascript"><span class="cm-number">100</span> <span class="cm-operator">==</span> <span class="cm-number">1e+2</span><span class="cm-number">0.01</span> <span class="cm-operator">==</span> <span class="cm-number">1e-2</span><span class="cm-number">12.34</span> <span class="cm-operator">==</span> <span class="cm-number">1.234e+1</span><span class="cm-number">0.000456</span> <span class="cm-operator">==</span> <span class="cm-number">4.56e-4</span></code></pre><p>The basic idea is that the "coefficient", to the <em>left</em> of "e" has its decimal place moved a number of places equal to the "exponent", to the <em>right</em> of "e". So:</p><pre><code>4.56e-4=======    4.56     ^ v moved left 40.0004561.234e+1========1.234 ^  v moved right 112.34</code></pre><p>That's all there is to it, really. But the problem is, JavaScript loves this scientific notation, even to the point that it can use it even when the number is in a manageable size. And there is no way of getting the original, full number as a string with any native method. And your job is to write one.</p><p>Write a method that can be called on any number that returns the number in all it's full, long-winded, memory-intensive string glory. ^^</p>
