<h2><a href=https://www.codewars.com/kata/58603c898989d15e9e000475/train/javascript target="_blank">Master your primes: sieve with memoization</a></h2><h3>5 kyu</h3><p>As most of you might know already, a prime number is an integer <code>n</code> with the following properties:</p><ul><li>it must be greater than 1</li><li>it must be divisible only by itself and 1</li></ul><p>And that's it: -15 or 8 are not primes, 5 or 97 are; pretty easy, isn't it?</p><p>Well, turns out that primes are not just a mere mathematical curiosity and are very important, for example, to allow a lot of cryptographic algorithms.</p><p>Being able to tell if a number is a prime or not is thus not such a trivial matter and doing it with some efficient algo is thus crucial.</p><p>There are already more or less efficient (or sloppy) katas asking you to find primes, but here I try to be even more zealous than other authors.</p><p>You will be given a preset array/list with the first few <code>primes</code>. And you must write a function that checks if a given number <code>n</code> is a prime looping through it and, possibly, expanding the array/list of known primes only if/when necessary (ie: as soon as you check for a <strong>potential prime which is greater than a given threshold for each</strong> <code>n</code>, stop).</p><h1 id="memoization">Memoization</h1><p>Storing precomputed results for later re-use is a very popular programming technique that you would better master soon and that is called <a href="https://en.wikipedia.org/wiki/Memoization" data-turbolinks="false" target="_blank">memoization</a>; while you have your wiki open, you might also wish to get some info about the <a href="https://en.wikipedia.org/wiki/Sieve_of_Eratosthenes" data-turbolinks="false" target="_blank">sieve of Eratosthenes</a> (one of the few good things I learnt as extra-curricular activity in middle grade) and, to be even more efficient, you might wish to consider <a href="https://medium.com/@lcthornhill/why-does-the-6-iteration-method-work-for-testing-prime-numbers-ba6176f58082#.dppj0il3a" data-turbolinks="false" target="_blank">an interesting reading on searching from prime from a friend of mine</a> [she thought about an explanation all on her own after an evening event in which I discussed primality tests with my guys].</p><p>Yes, you will be checked even on that part. And you better be <strong>very</strong> efficient in your code if you hope to pass all the tests ;)</p><p><strong>Dedicated to my trainees that worked hard to improve their skills even on a holiday: good work guys!</strong></p><p><strong>Or should I say "girls" ;)? <a href="https://www.codewars.com/users/aghh1504" data-turbolinks="false" target="_blank">Agata</a>, <a href="https://www.codewars.com/users/lisowska" data-turbolinks="false" target="_blank">Ania</a> <a href="https://www.codewars.com/users/deedeeh" data-turbolinks="false" target="_blank">Dina</a> and (although definitely not one of my trainees) special mention to <a href="https://www.codewars.com/users/nbeck" data-turbolinks="false" target="_blank">NaN</a></strong></p>
