<h2><a href=https://www.codewars.com/kata/5a02cf76c9fc0ee71d0000d5/train/javascript target="_blank">Getting along with Bernoulli's numbers</a></h2><h3>5 kyu</h3><p>Let us define a function <code>f</code>such as:</p><ul><li>(1) for k positive odd integer &gt; 2 :</li></ul><p><img title="\bg_green f(k) = \sum_{n=1}^{nb}1/n^{k}" src="https://latex.codecogs.com/gif.latex?\bg_green&amp;space;f(k)&amp;space;=&amp;space;\sum_{n=1}^{nb}1/n^{k}"></p><ul><li>(2) for k positive even integer &gt;= 2 :</li></ul><p><img title="\bg_green f(k) = 1/2 |B_k| (2\pi)^{k} / k!" src="https://latex.codecogs.com/gif.latex?\bg_green&amp;space;f(k)&amp;space;=&amp;space;1/2&amp;space;|B_k|&amp;space;(2\pi)^{k}&amp;space;/&amp;space;k!"></p><ul><li>(3) for k positive integer &gt; 1 :</li></ul><p><img title="f(-k) = (-1)^k*B_{k+1}/(k + 1))" src="https://latex.codecogs.com/gif.latex?\bg_green&amp;space;f(-k)&amp;space;=&amp;space;(-1)^k*B_{k+1}/(k&amp;space;+&amp;space;1))"></p><p>where <code>|x|</code> is <code>abs(x)</code> and B<font size="1">k</font> the k<sup>th</sup> Bernoulli number.</p><p><code>f</code> is not defined for <code>0, 1, -1</code>. These values will not be tested.</p><h4 id="guidelines-for-bernoulli-numbers">Guidelines for Bernoulli numbers:</h4><p><a href="https://en.wikipedia.org/wiki/Bernoulli_number" data-turbolinks="false" target="_blank">https://en.wikipedia.org/wiki/Bernoulli_number</a></p><p><a href="http://mathworld.wolfram.com/BernoulliNumber.html" data-turbolinks="false" target="_blank">http://mathworld.wolfram.com/BernoulliNumber.html</a></p><p><a href="https://www.codewars.com/kata/bernoulli-numbers-1" data-turbolinks="false" target="_blank">https://www.codewars.com/kata/bernoulli-numbers-1</a></p><p>There is more than one way to calculate them. You can make Pascal triangle and then with the basic formula below generate all Bernoulli numbers.</p><p>1 + 2B = 0 ... gives ...  B = - 1/2</p><p>1 + 3B + 3B = 0 ... gives ...    B        = 1/6</p><p>1 + 4B + 6B + 4B = 0 ... gives ... B = 0</p><p>1 + 5B + 10B + 10B + 5B = 0 ... gives ... B = - 1/30</p><p>... and so on</p><h4 id="task">Task</h4><p>Function <code>series(k, nb)</code> returns (1), (2) or (3) where <code>k</code> is the <code>k</code> parameter of <code>f</code> and <code>nb</code> the upper bound in the summation of (1). <code>nb</code> is of no use for (2) and (3) but for the sake of simplicity it will always appear in the tests even for cases (2) and (3) with a value of <code>0</code>.</p><pre><code>ExamplesS(2, 0) = 1.644934066848224....S(3, 100000) = 1.20205690310973....S(4, 0) = 1.08232323371113.....S(-5, 0) = -0.003968253968....S(-11, 0) = 0.02109279609279....</code></pre><h4 id="notes">Notes</h4><ul><li>For Java, C#, C++: <code>k</code> should be such as <code>-27 &lt;= k &lt;= 20</code>; otherwise <code>-30 &lt;= k &lt;= 30</code> and be careful of 32-bit integer overflow.</li><li>Translators are welcome.</li></ul>
