Both URLs are stored in a database. It is important that each long URL is assigned a unique short URL.</p><p>If a user then calls up the short URL, the database is checked to see which long URL belongs to this short URL and you are redirected to the original/long URL.</p><p><strong>Important Note:</strong> Some URLs such as <code>www.google.com</code> are used very often. It can happen that two users want to shorten the same URL, so you have to check if this URL has been shortened before to save memory in your database.</p><h2 id="task">Task</h2><p>Write a class with two methods, <code>shorten</code> and <code>redirect</code></p><h3 id="url-shortener">URL Shortener</h3><p>Write a method <code>shorten</code>, which receives a long URL and returns a short URL starting with <code>short.ly/</code>, consisting only of <code>lowercase letters</code> (and one dot and one slash) and max length of <code>13</code>.</p><p><strong>Note:</strong> <code>short.ly/</code> is not a valid short URL.</p><h3 id="redirect-url">Redirect URL</h3><p>Write a method <code>redirect</code>, which receives the shortened URL and returns the corresponding long URL.</p><h2 id="performance">Performance</h2><p>There are <code>475_000</code> random tests.You don't need a complicated algorithm to solve this kata, but iterating each time through the whole database to check if a URL was used before or generating short URLs based on randomness, won't pass.</p><p><strong>GOOD LUCK AND HAVE FUN!</strong></p>
