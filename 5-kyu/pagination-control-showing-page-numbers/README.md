<h2><a href=https://www.codewars.com/kata/57a7cb5f53ba3350c60000e7/train/javascript target="_blank">Pagination control - Showing page numbers</a></h2><h3>5 kyu</h3><p>You are building a pagination control that will meet the following requirements:</p><ul><li>Should always show the first and last page</li><li>Should show N pages to navigate before and after the current page (not counting the first and last)</li><li>If there is a single page it should return an empty array</li></ul><p>For example:</p><ul><li>Total pages: 10</li><li>Current page: 5</li><li>Items before/after: 2</li><li>Expected: <code>[1,</code> (always) <code>3,4</code> (2 before) <code>,5,</code> (center) <code>6,7,</code> (2 after) <code>10]</code> (always)</li></ul><p>Another example:</p><ul><li>Total pages: 36</li><li>Current page: 1</li><li>Items before and after: 5</li><li>Expected <code>[1</code> (always) <code>2,3,4,5,6</code> (5 before) <code>,7,</code> (center) <code>8,9,10,11,12,</code> (5 after) <code>36]</code> (always)</li></ul><p><img alt="img1" src="https://i.imgur.com/EKIyxc9.png"></p><p>Current page 7:</p><p><img alt="img2" src="https://i.imgur.com/uVmc161.png"></p><p>Current page 8:</p><p><img alt="img3" src="https://i.imgur.com/O6AlTL8.png"></p><p>The "center" is one way to look at it. The important thing is that the number of pages keep consistent so the user will have a nice experience.</p><p>When the current page is, for example, 1 if you subtract 5 from the "center" you would get negative values. Negative pages are invalid, so you have to compensate these pages to the right side. The same goes when your page goes over the total pages, you have to compensate left.</p><p>Complete the function so it returns an array with the items for you to render on screen!</p><p>You can use the pre-loaded <code>printPages</code> to see how your result looks like.</p>
