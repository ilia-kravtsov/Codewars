<h2><a href=https://www.codewars.com/kata/52fff3f4aff4559096000917/train/javascript target="_blank">Default configuration</a></h2><h3>5 kyu</h3><p>A common pattern is for a function to take a configuration object as a parameter, implementing defaults or throwing errors for any properties missing from the configuration.</p><p>One way to do this is like:</p><pre><code>function send(config) {  if(!config.url) throw "No URL!";  config.method = config.method || "POST";  config.data = config.data || {}  Network.send(config.url, config.data, config.method);}</code></pre><p>But this can get cumbersome for a large number of arguments, and the "||" operator doesn't work correctly for parameters that are allowed to be falsey.</p><p>Your task is to create a <code>defaults(obj, def)</code> function which creates a new object with <code>obj</code>'s properties, falling back on the properties of <code>def</code> when they aren't in <code>obj</code>.</p><p>You must also implement the <code>mandatory(err)</code> function. This will return an object that tells defaults to throw <code>err</code> if the property doesn't exist.</p><p>In other words, if <code>mandatory("foo")</code> is given as the default value for the key <code>bar</code>, and the user tries pass a config option that does not define <code>bar</code>, then <code>defaults</code> should throw "foo" when called.</p><p>This function would be used like so:</p><pre><code>function send(config) {  config = defaults(config, {    url: mandatory("No URL!"),    method: "POST",    data: {}  });  Network.send(config.url, config.data, config.method);}</code></pre><p>Your function should not modify the original object, since the user may want to reuse the same config object multiple times.</p>
